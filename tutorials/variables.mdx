# O'zgaruvchilar

Ko'p hollarda JavaScript ilovalari ma'lumotlar bilan ishlashiga to'g'ri keladi. Masalan:

1. Online do'kon - ma'lumot sotilayotgan mahsulotlar va xarid savatchasi haqida bo'lishi mumkin.
2. Chat ilovasi - ma'lumot foydalanuvchilar, xabarlar va boshqalar haqida bo'lishi mumkin.

O'zgaruvchilar esa aynan shu ma'lumotlarni saqlash uchun ishlatiladi.

## O'zgaruvchi

O'zgaruvchi ([variable](<https://en.wikipedia.org/wiki/Variable_(computer_science)>)) bu ma'lumotni biror nom bilan saqlash usulidir.

JavaScript'da o'zgaruvchi yaratish uchun `let` kalit so'zdan (keyword) foydalaniladi.

Quyidagi satr orqali `message` nomli o'zgaruvchi yaratiladi (yoki _e'lon qilinadi_):

```javascript
let message;
```

Endi, biz bu o'zgaruvchiga `=` (tenglik) operatori orqali qiymat berishimiz mumkin:

```javascript
let message;

message = 'Hello!'; // 'Hello!' satrni message o'zgaruvchisida saqlash
```

`'Hello!'` satri endi xotiraning `message` o'zgaruvchisiga ajratilgan joyida saqlanyapti.
Bu qiymatni kodning qolgan qismlarida tegishli o'zgaruvchi nomi bilan ola olamiz.

```javascript
let message;
message = 'Hello!';

alert(message); // message o'zgaruvchisida saqlangan qiymatni ko'rsatadi
```

Qisqartirish maqsadida o'zgaruvchi e'lon qilish va unga qiymat berishni bitta satrda quyidagicha yozishimiz ham mumkin:

```javascript
let message = 'Hello!';

alert('Hello!'); // Hello!
```

Bir nechta o'zgaruvchilarni ham bitta satrda e'lon qilishimiz mumkin:

```javascript
let user = 'John',
  age = 25,
  message = 'Hello';
```

Bu usul qisqa bo'lgani bilan odatda tavsiya etilmaydi.
Bunday e'lon qilish sizning kodingizning oson o'qilishi va tushunarliligiga putr yetkazishi mumkin.

Har bir o'zgaruvchiga alohida satr ajratish uzun ko'ringani bilan birga o'qishga osonroqdir:

```javascript
let user = 'John';
let age = 25;
let message = 'Hello';
```

Ba'zi dasturchilar o'zgaruvchilarni quyidagicha ham e'lon qiladilar:

```javascript
let user = 'John',
  age = 25,
  message = 'Hello';
```

...yoki avval vergulni yozish orqali:

```javascript
let user = 'John'
  ,age = 25,
  ,message = 'Hello';
```

Texnik jihatdan barchasi bir xil vazifani bajaradi. Qaysi usulni tanlash esa shaxsiy did va estetikaga bog'liq.

> **`let` o'rnida `var`**
>
> Nisbatan eski kodlarda `let` kalit so'zi o'rnida `var` ishlatilishiga guvoh bo'lishingiz mumkin.
>
> ```javascript
> var message = 'Hello';`
> ```
>
> `var` kalit so'zi ham `let` bilan _deyarli_ bir xil tarzda ishlaydi. Shunday bo'lsa ham
> ular orasida farqlar mavjud. `var` va `let` kalit so'zlari farqlarini quyidagi bo'limda ko'rib chiqamiz: [Eski "var"](#)

## Hayotiy qiyoslash

Biz "o'zgaruvchi" tushunchasiga berilgan ma'lumotni saqlash uchun mo'ljallangan nomlangan quti sifatida qarashimiz mumkin.

Masalan, `message` o'zgaruvchisi `"Hello!"` qiymatni saqlash uchun mo'ljallangan "message" nomli qutidir.

> IMAGE 1

Biz qutiga ixtiyoriy qiymatni qo'yishimiz mumkin.

Qo'yilgan qiymatni esa istalgancha almashtirishimiz mumkin.

```javascript
let message;

message = 'Hello!';

message = 'World!'; // qiymat o'zgardi

alert(message);
```

Qiymat o'zgartirilganda qutidagi eski qiymat o'chib ketadi.

> IMAGE 2

Bundan tashqari, biz ikkita o'zgaruvchi e'lon qilib,
birining qiymatini boshqasining qiymati bilan almashtirishimiz ham mumkin.

```javascript
let message;

// hello'dan 'Hello world!' ni message'ga ko'chirish
message = hello;

// Endi ikkala o'zgaruvchi ham bir xil ma'lumotni saqlayapti
alert(hello); // Hello world!
alert(message); // Hello world!
```

> **Bir xil nomli ikkita o'zgaruvchi e'lon qilish xatolikka olib keladi**
>
> O'zgaruvchi faqat bir marta e'lon qilinishi kerak.
>
> ```javascript
> let message = 'This';
>
> // takrorlangan 'let' xatolikka olib keladi
> let message = 'That'; // SyntaxError: 'message' has already been declared
> ```

## O'zgaruvchini nomlash

JavaScript dasturlash tilida o'zgaruvchi nomlariga ikki xildagi cheklovlar mavjud:

1. Tanlangan nom faqat harflar, raqamlar yoki `$` va `_` belgilaridangina iborat bo'lishi;
2. Tanlangan nom raqam bilan boshlanmasligi;

To'g'ri qo'yilgan nomlarga misollar:

```javascript
let userName;
let test123;
```

Agar o'zgaruvchi nomi bir nechta so'zlardan iborat bo'lsa odatda [camelCase](https://en.wikipedia.org/wiki/Camel_case)
ko'rinishida yoziladi. Masalan, `myVeryLongName`.

Aytib o'tish joizki, `$` va `_` belgilarining ishlatilishi ham harflardan farq qilmaydi.
Masalan quyidagi o'zgaruvchi nomlari o'rinlidir:

```javascript
let $ = 1; // "$" nomli o'zgaruvchi e'lon qilindi
let _ = 2; // "_" nomli o'zgaruvchi e'lon qilindi

alert($ + _); // 3
```

Noto'g'ri qo'llangan o'zgaruvchi nomlariga quyidagilar misol bo'ladi:

```javascript
let 1a; // raqam bilan boshlanishi mumkin emas

let my-name; // '-' belgisi ishlatilishi mumkin emas
```

> O'zgaruvchi nomlarida katta va kichik harflar ahamiyatlidir.
> Masalan, `apple` va `AppLE` turli o'zgaruvchilar hisoblanadi.

> O'zgaruvchi nomlarida Lotin bo'lmagan alifbolardan ham foydalanish mumkin:
>
> ```javascript
> let имя = '...';
> let 我 = '...';
> ```
>
> Bu tarzda o'zgaruvchi e'lon qilish xato bo'lmasa ham an'anaga ko'ra Lotin alifbosidan foydalanish tavsiya etiladi.

**Band qilingan nomlar**

JavaScript dasturlash tilida [band qilingan nomlar ro'yxati](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Lexical_grammar#keywords) mavjud.
Ular dasturlash tili tomonidan ishlatilgani uchun biz, foydalanuvchi, tomonidan yaratiluvchi o'zgaruvchilarni nomlash uchun ishlatilishi mumkin emas.

Masalan `let`, `class`, `return` va `function` nolari banddir.

Quyidagi kod xatolik chiqaradi:

```javascript
let let = 5; // "let" nomli o'zgaruchi e'lon qilish mumkin emas
let return = 5; // "return" nomli o'zgaruchi e'lon qilish mumkin emas
```

**`use strict` bo'lmaganda qiymat berish**

Odatda, o'zgaruvchi ishlatilishidan oldin yaratilishi (e'lon qilinishi) kerak. Ammo, oldingi vaqtlarda o'zgaruvchini (`let` orqali e'lon qilmay turib) qiymat
berish orqali ham yaratish mumkin edi. Bu tarzda o'zgaruvchi yaratish hozir ham `use strict` ta'sirida bo'lmagan joylarda xatolik keltirib chiqarmaydi.

```javascript
// E'tibor bering: "use strict" mavjud emas

num = 5; // "num" nomli o'zgaruvchi yaratildi (agar allaqachon yaratilmagan bo'lsa)

alert(num); // 5
```

Bunday yozilish tavsiya etilmaydi va `use strict` muhitida xatolik keltirib chiqaradi:

```javascript
'use strict';

num = 5; // error: num is not defined
```

## O'zgarmaslar

O'zgarmas (konstanta) o'zgaruvchi yaratish uchun `let` o'rnida `const` kalit so'zi ishlatiladi.

```javascript
const myBirthday = '18.04.1982';
```

`const` orqali yaratilgan o'zgaruvchilar "o'zgarmas" yoki "konstanta" deb ataladi. Bu tipdagi o'zgaruvchilar qiymatini
yangilash xatolik keltirib chiqaradi (`let` dan farqli ravishda):

```javascript
const myBirthday = '18.04.1982';

myBirthday = '01.01.2001'; // Error! O'zgarmasga qayta qiymat berish mumkin emas
```

Dasturchi yaratilyotgan o'zgaruvchi kelgusida o'zgartirilmasligiga amin bo'lsa `const` orqali e'lon qilishi mumkin.
Bu orqali dasturchi jamoadagi qolgan a'zolarga ham bu haqida ma'lumot beradi.

**Bosh harfli o'zgarmaslar**

Eslab qolish qiyin va/yoki oldindan tayin bo'lgan qiymatlarni bosh harfli o'zgarmaslarga saqlab olish odat tusiga kirgan.

Bosh harfli o'zgarmaslar odatda alifbodagi bosh harflar va `_` belgisi bilan belgilanadi.

Masalan veb dasturlashdagi ranglarni eslab qolish va o'qiganda tushunish qiyin bo'lgani uchun quyidagi belgilashlarni amalga oshirish
mumkin:

```javascript
const COLOR_RED = '#F00';
const COLOR_GREEN = '#0F0';
const COLOR_BLUE = '#00F';
const COLOR_ORANGE = '#FF7F00';

let color = COLOR_ORANGE;
alert(color); // #FF7F00
```

Afzalliklari:

- `COLOR_ORANGE` nomi `"#FF7F00"` nomiga nisbatan osonroq eslab qolinadi.
- `COLOR_ORANGE` ga nisbatan `"#FF7F00"` ni yozishda xato yozib qo'yish ehtimoli kattaroq.
- Kod o'qilyotganda `COLOR_ORANGE` nomi ko'proq ma'lumot taqdim etadi.

Qachon barcha harflar bosh harf bilan yozilishi kerak?

Turli vaziyatlarda turli xil o'zgarmaslar bilan ishlashga to'g'ri keladi. Ba'zi o'zgarmaslar dastur ishlatilgan vaqtlarda
hisoblab olinadi va dastur davomida o'zgarishsiz qoladi, masalan:

```javascript
const pageLoadTime = /* veb sahifani yuklash uchun ketuvchi vaqt */;
```

Yuqoridagi misolda `pageLoadTime` dastur ishlatilishidan oldin bizga ma'lum emas. Ammo, dastur davomida o'zgarmasdan qoladi.
Aksincha, masalan, qizil rang uchun o'n oltilik kod dastur qanday bo'lishidan qat'iy nazar bizga ma'lum va doim o'zgarmasdir.
Shunday hollarda barchasi bosh harflardan iborat bo'lgan o'zgaruvchi nomlari ishlatilishi tavsiya etiladi.

**O'zgaruvchilarni to'g'ri nomlash**

O'zgaruvchilarni nomlashda tanlangan nomning ma'nosi aniq va oydin bo'lishi maqsadga muvofiqdir.

Ba'zi tavsiyalar:

- Odam tushunadigan `userName` yoki `shoppingCart` kabi jumlalar oqali nomlang.
- `a`, `b`, `c` kabi qisqa nomlardan qoching.
- Nomlarni maksimal darajada tushunarli va ma'noli qilib tanlang. Masalan `data` yoki `value` nomlari yaxshi tanlanmagan
  nomlar deyish mumkin. Chunki ular o'ta umumiy nomlardir.
- Nom tanlashdan oldin jamoa a'zolari bilan umumiy qoidalarni kelishib oling.

## Xulosa

Biz ma'lumot saqlash uchun o'zgaruvchilarni `let`, `var` va `const` kalit so'zlaridan foydalanishimiz mumkin.

- `let` - e'lon qilishning nisbatan yangi usuli.
- `var` - e'lon qilishning nisbatan eski usuli. Hozirgi dasturlarda bu kalit so'z deyarli ishlatilmaydi, lekin `var` va `let` nomining
  farqlari haqida [Eski "var"](#) bo'limida to'xtalib o'tamiz.
- `const` - e'lon qilishning `let` ga o'xshash usuli, biroq bu tarzda yaratilgan o'zgaruvchilar qiymatlari yangilana olmaydi.

O'zgaruvchi nomlari ular qanday ma'lumot saqlashini aniq va to'g'ri bildira olishi kerak.
